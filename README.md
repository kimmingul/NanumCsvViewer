# Nanum CSV Viewer

대용량 CSV 파일을 **빠르게 열고** 부드럽게 탐색하는 Windows 데스크톱 뷰어입니다.
1 GB가 넘는 CSV도 첫 페이지가 즉시 표시되며, 한글 인코딩(EUC-KR / CP949 / UTF-8)을 정확히 처리합니다.

> .NET 10 · Windows Forms · 단일 실행 파일(.exe) 배포 지원

---

## ✨ 주요 기능

- **대용량 즉시 열람** — 파일 전체를 메모리에 적재하지 않고, 바이트 오프셋 인덱스 + `DataGridView` 가상 모드로 보이는 행만 그때그때 읽습니다. 1 GB 파일도 첫 페이지가 1초 안에 뜹니다.
- **백그라운드 인덱싱 + 진행 표시** — 첫 페이지를 보여주면서 나머지를 백그라운드로 인덱싱하고, 하단 상태바에 **진행바 + %** 를 표시합니다. 완료되면 "준비 완료"와 함께 필터/정렬이 활성화됩니다.
- **한글 인코딩 자동 감지 + 수동 전환** — BOM 확인 → strict UTF-8 검증 → 실패 시 CP949(EUC-KR) 폴백. 상태바 우측의 인코딩 표시(신호등 왼쪽)를 클릭하거나 View ▸ 인코딩 메뉴로 즉시 바꿔 다시 디코딩할 수 있습니다(재인덱싱 불필요).
- **적응형 메모리** — RAM 예산(기본 1.5 GB) 이내면 원본 바이트를 메모리에 보관해 필터/정렬이 빠르고, 초과하면 자동으로 디스크 스캔 모드로 전환해 메모리를 일정하게 유지합니다.
- **검색 / 필터 / 정렬** — 스트리밍 "Find Next", 컬럼별/전체 필터, 컬럼 헤더 클릭 정렬. **헤더 Shift+클릭으로 다중 컬럼 정렬**(우선순위 표시)을 지원하며, 같은 값끼리는 원본 순서를 유지하는 **안정 정렬**입니다.
- **한눈에 보는 상태바** — 좌측에 **행 수 · 파일 용량**, 우측에 인코딩 위젯과 상태 표시등을 표시합니다.
- **조건 누적 필터(AND)** — 텍스트 필터에 더해, 선택한 셀의 값으로 즉시 필터를 거는 **`Filter by Cell`** 버튼/우클릭 메뉴를 제공합니다. 여러 조건이 **AND로 누적**되어 Excel처럼 점점 좁혀 볼 수 있습니다. 조건을 추가할 때는 전체가 아니라 **현재 결과만 다시 거르므로**(증분 필터) 2중·3중으로 갈수록 빨라집니다.
- **상태 표시등(신호등)** — 상태바 우측 끝의 색 점으로 상태를 한눈에: ⚪ 대기 · 🟡 불러오는 중 · 🟠 작업 중 · 🟢 준비 완료.
- **원본 행번호** — 행 헤더에 원본 파일 기준 행번호를 표시합니다. 필터·정렬해도 원래 위치를 추적할 수 있고, 헤더 폭은 드래그로 조절할 수 있습니다.
- **셀 내 줄바꿈 표시** — 따옴표로 감싼 필드 안의 줄바꿈을 여러 줄로 보여줍니다(대용량에서도 끊김 없이 셀 선택).
- **선택 항목 보기** — 그리드 위 셀 값 표시줄(높이 확장 가능)과, 토글 가능한 **우측 상세 패널**(선택한 행의 모든 컬럼을 컬럼명+값으로 표시, **F4**). 한 셀에 여러 줄이 담긴 데이터를 읽기에 좋습니다.

## 🧰 기술 스택

| 항목 | 내용 |
|---|---|
| 런타임 | .NET 10 (`net10.0-windows`) |
| UI | Windows Forms |
| CSV 엔진 | 외부 라이브러리 없이 직접 구현(SIMD 바이트 스캐너 + span 파서) |
| 인코딩 | `CodePagesEncodingProvider`로 CP949/EUC-KR(코드페이지 949) 지원 |

## 🚀 빌드 & 실행

요구 사항: [.NET 10 SDK](https://dotnet.microsoft.com/download)

```bash
# 빌드
dotnet build -c Release

# 실행
dotnet run --project NanumCsvViewer -c Release

# 테스트(CSV 파서·인덱서·인코딩 감지 등 단위/통합 테스트)
dotnet test
```

## 📦 단일 실행 파일 만들기

.NET 설치 없이 어디서나 실행되는 단일 `.exe`(자체 포함 + 압축)를 만듭니다.

```bash
dotnet publish NanumCsvViewer/NanumCsvViewer.csproj -p:PublishProfile=win-x64-singlefile
```

산출물: `NanumCsvViewer/bin/Release/publish/NanumCsvViewer.exe` (약 50 MB).
Visual Studio에서는 게시 시 `win-x64-singlefile` 프로필을 선택하면 됩니다.

## 📖 사용법

1. **File ▸ Open**(또는 툴바 Open)으로 `.csv` / `.txt` 파일을 엽니다.
2. 하단 진행바가 끝나면(또는 진행 중에도) 자유롭게 스크롤합니다.
3. **인코딩**: 상태바 우측(신호등 왼쪽)의 인코딩 표시를 클릭해 바꿉니다(자동 감지가 틀린 경우). View ▸ 인코딩 메뉴로도 가능합니다.
4. **Find**: 검색어 입력 후 Enter → 다음 일치로 이동.
5. **Filter**: 컬럼 선택 + 검색어 → Apply. 셀을 선택한 뒤 **`Filter by Cell`**(또는 우클릭 ▸ "이 셀 값으로 필터")로 그 값 조건을 추가하면 기존 필터에 **AND로 누적**됩니다. **Clear**로 전체 해제.
6. **정렬**: 컬럼 헤더 클릭(다시 클릭 시 내림차순).
7. **상세 패널**: 기본으로 우측에 표시되며, 툴바 `Detail Panel` 버튼 또는 **F4**로 켜고 끌 수 있습니다(폭 드래그 확장). 선택한 행 전체를 컬럼명+값으로 보여줍니다.
8. **상태 표시등**: 상태바 우측 끝 색 점으로 진행/준비 상태를 확인합니다(🟢 준비 완료).

## ⌨️ 단축키

| 단축키 | 기능 |
|---|---|
| `Ctrl+O` | 파일 열기 |
| `Ctrl+Q` | 종료 |
| `Ctrl+F` | 검색창으로 포커스 이동 |
| `F3` / `Enter`(검색창) | 다음 찾기 |
| `Enter`(필터창) | 필터 적용 |
| `Ctrl+B` | 선택한 셀 값으로 필터(AND 누적) |
| `Ctrl+Shift+L` | 필터 해제 |
| `Ctrl+Shift+S` | 정렬 해제 |
| `F4` | 상세 패널 토글 |

> 모든 기능은 **메뉴(File · Edit · View · Help)** 와 **툴바** 양쪽에서 접근할 수 있습니다.

### 정렬

- **단일 컬럼**: 컬럼 헤더 클릭(다시 클릭 시 오름↔내림 토글), 또는 `Sort ▲ / ▼` 툴바 버튼·Edit 메뉴(현재 셀의 열 기준).
- **다중 컬럼**: 헤더를 **Shift+클릭**하면 정렬 차수가 누적됩니다(1차, 2차, 3차…). 예: `이름` 클릭 후 `나이` Shift+클릭 → 이름 오름차순, 동률이면 나이로 정렬. 활성 컬럼에는 정렬 화살표가, 다중일 때는 우선순위 번호가 헤더에 표시됩니다.
- 같은 키 값을 가진 행끼리는 **원본 파일 순서가 유지**됩니다(안정 정렬). 숫자처럼 보이는 컬럼은 수치 기준으로 정렬합니다.
- 필터 조건을 **바꾸면** 정렬이 초기화됩니다(필터를 더 좁히는 경우는 순서 유지).

## ⚙️ 동작 원리(요약)

핵심은 "전부 읽지 않는다"입니다.

1. 파일을 한 번 순차 스캔하며 각 레코드의 **시작 바이트 오프셋**만 인덱싱합니다.
2. 화면에 보이는 행만 오프셋으로 찾아 읽고, 선택된 인코딩으로 디코딩·파싱해 LRU 캐시에 둡니다.
3. EUC-KR/CP949/UTF-8에서는 줄바꿈·구분자·따옴표 바이트가 멀티바이트 문자 중간에 나타나지 않으므로, 바이트 단위 스캔이 인코딩과 무관하게 안전합니다(따옴표 안 줄바꿈도 정확히 처리).

## ⚠️ 제한 사항

- 대용량 고속 모드는 **CSV/TXT 전용**입니다(Excel `.xls/.xlsx` 미지원).
- **UTF-16/UTF-32**는 대용량 모드에서 지원하지 않습니다(BOM 감지 시 안내).
- 매우 느린 HDD에서 디스크 모드(>약 1.5 GB)일 때 스크롤에 미세한 지연이 있을 수 있습니다.
