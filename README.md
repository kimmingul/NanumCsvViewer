# Nanum CSV Viewer

대용량 CSV 파일을 **빠르게 열고** 부드럽게 탐색하는 Windows 데스크톱 뷰어입니다.
1 GB가 넘는 CSV도 첫 페이지가 즉시 표시되며, 한글 인코딩(EUC-KR / CP949 / UTF-8)을 정확히 처리합니다.

> .NET 10 · Windows Forms · 단일 실행 파일(.exe) 배포 지원

---

## ✨ 주요 기능

- **대용량 즉시 열람** — 파일 전체를 메모리에 적재하지 않고, 바이트 오프셋 인덱스 + `DataGridView` 가상 모드로 보이는 행만 그때그때 읽습니다. 1 GB 파일도 첫 페이지가 1초 안에 뜹니다.
- **백그라운드 인덱싱 + 진행 표시** — 첫 페이지를 보여주면서 나머지를 백그라운드로 인덱싱하고, 하단 상태바에 **진행바 + %** 를 표시합니다. 완료되면 "준비 완료"와 함께 필터/정렬이 활성화됩니다. 인덱서는 **벡터화 바이트 스캔(`SearchValues`)** 과 **읽기·스캔 겹치기(더블 버퍼링)** 로 동작해, 일반 NVMe에서 인덱싱 처리량이 **~2 GB/s**에 이릅니다.
- **한글 인코딩 자동 감지 + 수동 전환** — BOM 확인 → strict UTF-8 검증 → 실패 시 CP949(EUC-KR) 폴백. 상태바 우측의 인코딩 표시(신호등 왼쪽)를 클릭하거나 View ▸ 인코딩 메뉴로 즉시 바꿔 다시 디코딩할 수 있습니다(재인덱싱 불필요).
- **적응형 메모리** — RAM 예산(기본 1.5 GB) 이내면 원본 바이트를 메모리에 보관해 필터/정렬이 빠르고, 초과하면 자동으로 디스크 스캔 모드로 전환해 메모리를 일정하게 유지합니다.
- **검색 / 필터 / 정렬** — 스트리밍 "Find Next", 컬럼별/전체 필터, 컬럼 헤더 클릭 정렬. **헤더 Shift+클릭으로 다중 컬럼 정렬**(우선순위 표시)을 지원하며, 같은 값끼리는 원본 순서를 유지하는 **안정 정렬**입니다.
- **한눈에 보는 상태바** — 좌측에 **행 수 · 파일 용량**, 우측에 인코딩 위젯과 상태 표시등을 표시합니다.
- **조건 누적 필터(AND)** — 텍스트 필터에 더해, 선택한 셀의 값으로 즉시 필터를 거는 **`Filter by Cell`** 버튼/우클릭 메뉴를 제공합니다. 여러 조건이 **AND로 누적**되어 Excel처럼 점점 좁혀 볼 수 있습니다. 조건을 추가할 때는 전체가 아니라 **현재 결과만 다시 거르므로**(증분 필터) 2중·3중으로 갈수록 빨라집니다.
- **상태 표시등(신호등)** — 상태바 우측 끝의 색 점으로 상태를 한눈에: ⚪ 대기 · 🟡 불러오는 중 · 🟠 작업 중 · 🟢 준비 완료.
- **원본 행번호** — 행 헤더에 원본 파일 기준 행번호를 표시합니다. 필터·정렬해도 원래 위치를 추적할 수 있고, 헤더 폭은 드래그로 조절할 수 있습니다.
- **셀 내 줄바꿈 표시** — 따옴표로 감싼 필드 안의 줄바꿈을 여러 줄로 보여줍니다(대용량에서도 끊김 없이 셀 선택).
- **선택 항목 보기** — 그리드 위 셀 값 표시줄(높이 확장 가능)과, 토글 가능한 **우측 상세 패널**(선택한 행의 모든 컬럼을 컬럼명+값으로 표시, **F4**). 한 셀에 여러 줄이 담긴 데이터를 읽기에 좋습니다.
- **다국어(영어·한국어)** — 기본은 영어이며, Windows 표시 언어가 한국어이면 자동으로 한국어로 시작합니다. **View ▸ Language**에서 즉시 전환할 수 있고, 선택은 저장됩니다.
- **라이트/다크 테마** — 툴바 우측의 토글 버튼으로 전환합니다. 첫 실행 시 **Windows 시스템 테마**를 따르고, 변경한 테마는 저장되어 다음 실행에 복원됩니다.
- **사용법 도움말** — **Help ▸ How to Use**(F1)에서 주요 기능과 단축키를 한 화면에서 안내합니다.
- **전용 앱 아이콘** — 작업표시줄·창 제목·설치 관리자·제거 항목에 일관된 아이콘이 적용됩니다.

### ✨ v1.8.0 신규 — 타입 인식 필터링·패싯 분석

- **컬럼 타입 확장** — 기존 `INT/FLT/DATE/BOOL/CAT/STR`에 더해 **`DateTime`·`Time`·`Identifier`**를 자동 추론합니다. `2024-01-01 14:30`은 DateTime, `14:30:00`은 Time으로 구분하고, `yyyyMMdd` 컴팩트 날짜는 헤더와 무관하게 날짜로 인식합니다(연도 1900~2100 가드). **식별자**(선행 0 숫자·`id`/`번호`/`코드` 헤더)는 수량이 아니므로 평균 등 무의미한 수치 통계를 자동 억제합니다.
- **전 타입 헤더 필터** — `Empty`를 제외한 모든 컬럼에 타입별 필터를 제공합니다: 숫자·시간계열은 **범위(min~max, 시각 정밀도 인식)**, 범주·불리언은 **체크박스**, 문자열·식별자는 **텍스트 술어**(포함·일치·시작·끝·정규식·빈값·목록 붙여넣기). 고유값이 적은 숫자 컬럼은 범위 대신 체크박스로 폴백합니다.
- **활성 필터 칩 바** — 모든 활성 조건을 그리드 위 칩으로 표시하고, 각 칩의 **✕로 개별 제거**·**클릭으로 재편집**합니다. 조건이 둘 이상이면 **모두(AND)/하나라도(OR) 결합 토글**을 제공하며, 결합 방식은 저장된 뷰에 함께 저장됩니다. 중첩 부울 `(A AND B) OR C`는 고급 식 필터로 표현할 수 있습니다.
- **패싯 분석 패널(F6)** — 우측 패널에서 컬럼별 분포를 한눈에: 숫자는 히스토그램, 그 외는 상위 값 빈도를 가로 막대로 표시합니다. 막대를 클릭하면 그 구간/값으로 즉시 필터되고, 필터가 바뀌면 분포가 다시 계산됩니다(**크로스필터링**).

### ✨ v1.7.0 신규 — 분석·탐색·생산성

- **고급 검색** — 평문(contains)에 더해 정규식(`/패턴/` 또는 `regex:`)과 퍼지(`fuzzy:`) 검색을 지원합니다.
- **표현식 필터** — `age > 30 AND city = "서울"` 처럼 `AND/OR/괄호`와 비교 연산을 쓰는 고급 필터(Edit ▸ Advanced Filter).
- **헤더 타입 배지** — 인덱싱이 끝나면 컬럼명 옆에 추론 타입(`INT/FLT/DATE/BOOL/CAT/STR`)을 컬러 배지로 표시(보기 메뉴·툴바에서 켜고 끄기).
- **헤더 필터** — 범주형/날짜 컬럼 헤더 우측의 깔때기로 값 체크박스(검색·개수)·날짜 범위 필터를 적용합니다(정렬과 분리).
- **내보내기** — 현재 보기(필터·정렬·표시 컬럼)를 **CSV / Markdown / JSON / HTML**로 스트리밍 내보내기.
- **분석** — 수치 분포(분위·히스토그램), 날짜 히스토그램(일/주/월/년), 중복 탐지, 그룹별 집계(Count/Sum/Mean/… ).
- **통계** — 상관(Pearson/Spearman), 독립·대응 t검정, 카이제곱 검정(정확한 p값·신뢰구간).
- **피벗 빌더 + 차트** — 행/열/값/필터에 필드를 배정하는 엑셀식 피벗(다중 측정·날짜 그룹핑·합계)과 **외부 라이브러리 없는 GDI+ 차트**(막대/묶은막대/누적/꺾은선). **피벗** 메뉴에서 표/차트로 진입.
- **엑셀식 셀 선택·복사** — 드래그 사각 선택·`Shift`/`Ctrl` 선택, `Ctrl+C`로 전체 값 TSV 복사, 우클릭으로 행/열 전체 복사, 상세 패널 TEXT/JSON 복사.
- **저장된 뷰** — 필터·정렬·숨김 컬럼·검색·컬럼 필터를 파일별로 저장/복원.
- **가져오기 편의** — 클립보드(CSV 텍스트·경로·`file://`)·드래그앤드롭으로 바로 열기.
- **성능 대시보드 · 영속 인덱스 캐시** — 처리량·저장 모드 등을 한눈에, 디스크 모드 재열기 시 인덱싱 생략.
- **행으로 이동(Ctrl+G) · 컬럼 숨기기** — 원본 행번호로 점프, 컬럼 표시/숨김 체크리스트.

```bash
# 빌드
dotnet build -c Release

dotnet run --project NanumCsvViewer -c Release

# 테스트(CSV 파서·인덱서·인코딩 감지 등 단위/통합 테스트)
dotnet test
```

## 📦 단일 실행 파일 만들기

릴리즈는 **Portable**과 **Install** 두 가지로 배포합니다(둘 다 프레임워크 의존 — 앱에 .NET 미포함).

| 산출물 | 형태 | 요구사항 |
|---|---|---|
| **Portable** | `…-portable.exe` — 설치 없이 실행하는 단일 exe | [.NET 10 Desktop Runtime (x64)](https://dotnet.microsoft.com/download/dotnet/10.0) (없으면 Windows가 안내) |
| **Install** | `…-setup.exe` — 시작 메뉴 등록·제거 지원 설치 관리자 | 설치 중 .NET 10을 점검해 **없으면 자동 설치** |

앱 빌드는 `win-x64-framework` 게시 프로필을 사용하고, 설치 관리자는 `installer/NanumCsvViewer.iss`(Inno Setup)로 만듭니다.
(.NET을 포함한 자체 포함본이 필요하면 `win-x64-singlefile` 프로필로 별도 게시할 수 있습니다.)

### 릴리즈 + 코드 서명

`scripts/release.ps1`이 게시 → **앱 exe 서명** → Portable 복사 → 설치 관리자 컴파일 → **setup 서명** → **설치된 exe 서명 검증**(임시 무인 설치·제거) → GitHub 릴리즈까지 수행합니다.

- 기본적으로 **x64·arm64 두 아키텍처를 모두** 빌드·서명하고 한 릴리즈에 함께 업로드합니다. 한쪽만 만들려면 `-Arch x64` 또는 `-Arch arm64`.
- Portable·setup·**설치되는 앱 exe** 모두 SafeNet 토큰의 EV 인증서로 서명·타임스탬프됩니다. (페이로드를 먼저 서명한 뒤 패키징하므로, Inno Setup이 바이트 보존하여 설치된 exe의 서명이 유지됩니다.)
- `-p:Version`으로 바이너리 파일버전을 릴리즈 버전과 자동 일치시킵니다.
- 서명 시 토큰 PIN 프롬프트가 뜹니다(연속 서명은 SafeNet single-logon 권장). 동일 버전 재실행 시 기존 릴리즈에 자산을 덮어씁니다.

```powershell
# 토큰 연결 상태에서 실행(코드사이닝 인증서 자동 선택)
.\scripts\release.ps1 -Version 1.8.0

# 빌드/컴파일만(서명·검증·릴리즈 생략)
.\scripts\release.ps1 -Version 1.8.0 -SkipSign -SkipRelease
```

요구: .NET SDK, **[Inno Setup 6.3+](https://jrsoftware.org/isdl.php)** (`winget install JRSoftware.InnoSetup`), Windows SDK(`signtool`), SafeNet Authentication Client, [GitHub CLI](https://cli.github.com/)(`gh auth login`).

## 📖 사용법

1. **File ▸ Open**(또는 툴바 Open)으로 `.csv` / `.txt` 파일을 엽니다.
2. 하단 진행바가 끝나면(또는 진행 중에도) 자유롭게 스크롤합니다.
3. **인코딩**: 상태바 우측(신호등 왼쪽)의 인코딩 표시를 클릭해 바꿉니다(자동 감지가 틀린 경우). View ▸ 인코딩 메뉴로도 가능합니다.
4. **Find**: 검색어 입력 후 Enter → 다음 일치로 이동.
5. **Filter**: 컬럼 선택 + 검색어 → Apply. 셀을 선택한 뒤 **`Filter by Cell`**(또는 우클릭 ▸ "이 셀 값으로 필터")로 그 값 조건을 추가하면 기존 필터에 **AND로 누적**됩니다. **Clear**로 전체 해제.
6. **정렬**: 컬럼 헤더 클릭(다시 클릭 시 내림차순).
7. **상세 패널**: 기본으로 우측에 표시되며, 툴바 `Detail Panel` 버튼 또는 **F4**로 켜고 끌 수 있습니다(폭 드래그 확장). 선택한 행 전체를 컬럼명+값으로 보여줍니다.
8. **상태 표시등**: 상태바 우측 끝 색 점으로 진행/준비 상태를 확인합니다(🟢 준비 완료).

### 정렬

- **단일 컬럼**: 컬럼 헤더 클릭(다시 클릭 시 오름↔내림 토글), 또는 `Sort ▲ / ▼` 툴바 버튼·Edit 메뉴(현재 셀의 열 기준).
- **다중 컬럼**: 헤더를 **Shift+클릭**하면 정렬 차수가 누적됩니다(1차, 2차, 3차…). 예: `이름` 클릭 후 `나이` Shift+클릭 → 이름 오름차순, 동률이면 나이로 정렬. 활성 컬럼에는 정렬 화살표가, 다중일 때는 우선순위 번호가 헤더에 표시됩니다.
- 같은 키 값을 가진 행끼리는 **원본 파일 순서가 유지**됩니다(안정 정렬). 숫자처럼 보이는 컬럼은 수치 기준으로 정렬합니다.
- 필터 조건을 **바꾸면** 정렬이 초기화됩니다(필터를 더 좁히는 경우는 순서 유지).

## ⚙️ 동작 원리(요약)

핵심은 "전부 읽지 않는다"입니다.

1. 파일을 한 번 순차 스캔하며 각 레코드의 **시작 바이트 오프셋**만 인덱싱합니다.
2. 화면에 보이는 행만 오프셋으로 찾아 읽고, 선택된 인코딩으로 디코딩·파싱해 LRU 캐시에 둡니다.
3. EUC-KR/CP949/UTF-8에서는 줄바꿈·구분자·따옴표 바이트가 멀티바이트 문자 중간에 나타나지 않으므로, 바이트 단위 스캔이 인코딩과 무관하게 안전합니다(따옴표 안 줄바꿈도 정확히 처리).

## ⚠️ 제한 사항

- 대용량 고속 모드는 **CSV/TXT 전용**입니다(Excel `.xls/.xlsx` 미지원).
- **UTF-16/UTF-32**는 대용량 모드에서 지원하지 않습니다(BOM 감지 시 안내).
- 매우 느린 HDD에서 디스크 모드(>약 1.5 GB)일 때 스크롤에 미세한 지연이 있을 수 있습니다.
